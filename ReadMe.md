### 나만의 스타트업 팀으로 토스플레이스 application 개발 이라는 미션이 들어왔을때, Agents별로 업무를 분담해서 스펙 기반으로 일하는 오케스트레이션 프로젝트 .


#### 예시
유저 : 토스 플레이스 앱을 만들고 싶어

| **달미** | 태스크 매니저 | 
| **도산** | 아키텍트 |
| **용산** | 코드 작성자 |
| **철산** | 코드 수정자 |
| **영실** | 코드 리뷰어 |
| **사하** | UI/UX 디자이너 |
| **인재** | C++ 빌드 자동화 전문가 | 
| **지평** | 테스터 | 
| **인재** | DevOps | 

각 역활별로, Agent간의 업무 수행과 완료, 체크 등 작업 수행하는 방식에대한 검토를 해보았으면해.
오케스트레이션인데, 일을 할 때마다 일관되게 결과물들이 나오는 방법이 필요해.

1. 에이전트 워크 플로우 일관되게 작업자들이 업무를 수행하고 확인하는 방법의 일관성 검토
2. 프로젝트의 폴더 구성
3. 크로스 플랫폼 빌드시 빌드되는 폴더의 위치
4. 프로그램에 openspec이 설치가 되지 않았을때, openspec을 기반으로 시작하게 하기 위한 행동을 누가 할지, 
5. 프로그램이 갑자기 종료되어도 다시 복구되는 세션을 사용한 방법 결정
6. 깃 워크트리를 사용한 병렬 에이전트 시스템.
